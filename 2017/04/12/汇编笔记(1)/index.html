王爽汇编笔记(1)

发表于 2017-04-12 |

王爽的汇编语言走马观花的看了几遍了，学这个完全是兴趣驱动，记得以前用C++ Builder 编写C++时，每次报错时，
弹出一个cpu窗口，满屏的汇编，看得头皮发麻。当时就在想，弹这个窗口给我调试？Are you kidding me。
由于当时工作上也需要一点点汇编，但还是边学边忘，主要是太抽象。
但是了解汇编又是这么重要，这次我决定每看完一章把自己的所思所得记录下来，以便以后能快速的捡起来。
不管怎么说，学完汇编还是可以配合IDA，OD玩玩crack。

第一章 基础知识

  1. 机器指令和汇编指令是一一对应的，比如：
    机器码： 10100001 00000011 00000000
    对应的汇编指令： MOV AX,[3]
    含义： 传送3号单元的内容放入AX

  2. 不同的cpu对应不同的汇编指令集

  3. 存储器的存储单元是从0开始编号的
  4. 一个存储单元有8位，即一个字节
  5. 1Byte=8bit 1KB=1024B 1MB=1024KB

  6. CPU控制着一切的芯片（各种存储器），那么CPU是如何沟通芯片呢？
    存储单元的地址（地址信息） –地址总线
    器件的选择，读或写的命令（控制信息） –控制总线
    读或写的数据（数据信息） –数据总线

    假设一个cpu有10根地址总线，我们知道一根线用1或0表示，那么10根就是2的10方，
    即只能寻址到0-1023

    80386cpu的地址总线宽度为32，则内存地址空间为4G。
    这也说明以前用的xp系统内存最大为4G，多了都是浪费，因为xp都是32位的。

  7. RAM：随机存储器，必须带电存储
    ROM：只读存储器，关机后数据依然存在

  8. cpu把连接的各存储器件当作内存对待，把他们总的看作一个由若干存储单元组成的逻辑存储器，
    这就是我们所说的内存地址空间，汇编面对的就是内存地址空间。

第二章 寄存器

  1. 8086cpu有14个寄存器，AX,BX,CX,DX,SI,DI,SP,BP,IP,CS,SS,DS,ES,PSW.
    其中每个寄存器都是16位的,即2个字节或说1个字

  2. 其中AX,BX,CX,DX称为通用寄存器。以AX为例， 左8位为低8位（AL），右8位为高8位（AH）

  3. 物理地址=基础地址（段地址（SA）x16）+偏移地址(EA)
    注意这里的16是十进制，对应的16进制是10H。

    物理地址为什么要用这种基地址+偏移地址的表示法？
    是这样的，比如一个四位数，我怎么样用两张只能表示3位数的纸条表达呢。
    所以汇编采用了一个3位数x16+另一个3位数 = 一个四位数的表示法。

    汇编用这种表示法其实是有道理的，8086cpu 有20根地址总线，而他是16位架构的cpu，一次最多只能传输16位的地址，
    为了能访问2的20次方的地址长度，就出现了这种物理地址的表示法。当然地址的解析是内部的地址加法器完成。

  4. 将若干地址连续的内存单元看作一个段。
    那么段地址的起始地址就是基础地址。注意起始地址必是16的倍数，如果偏移地址是16位，那么最大寻址就是64KB

各个寄存器一览

8086 CPU 中寄存器总共为 14 个，且均为 16 位 。
即 AX，BX，CX，DX，SP，BP，SI，DI，IP，FLAG，CS，DS，SS，ES 共 14 个。
而这 14 个寄存器按照一定方式又分为了通用寄存器，控制寄存器和段寄存器。

通用寄存器：

AX，BX，CX，DX 称作为数据寄存器：

AX (Accumulator)：累加寄存器，也称之为累加器；
BX (Base)：基地址寄存器；
CX (Count)：计数器寄存器；
DX (Data)：数据寄存器；
SP 和 BP 又称作为指针寄存器：
SP (Stack Pointer)：堆栈指针寄存器；
BP (Base Pointer)：基指针寄存器；
SI 和 DI 又称作为变址寄存器：
SI (Source Index)：源变址寄存器；
DI (Destination Index)：目的变址寄存器；

控制寄存器：

IP (Instruction Pointer)：指令指针寄存器；
FLAG：标志寄存器；

段寄存器：

CS (Code Segment)：代码段寄存器；
DS (Data Segment)：数据段寄存器；
SS (Stack Segment)：堆栈段寄存器；
ES (Extra Segment)：附加段寄存器；

CS 和 IP

CS和IP指示了CPU当前要读取指令的地址。
即任意时刻，CPU将CS：IP指向的内容当做指令执行。

注意这句话的另一层含义，在内存中，指令和数据都是二进制，那CPU怎么知道内存中哪些是指令哪些是数据呢？
答案自然是凡是被CS：IP指向过的都当作指令执行。

CPU工作过程：
（1） 从CS：IP指向的内存单元读取指令，读取的指令进入指令缓冲器；
（2） IP=IP+所读取的指令长度，从而指向下一条指令；
（3） 执行指令。转到步骤（1），重复执行。

如何修改CS和IP

在8086CPU中，程序员是无法直接通过比如mov等指令来修改cs和ip。而是：
jmp 段地址：偏移地址
jmp 某一合法的寄存器

能够修改cs和ip的都叫转移指令

Debug常用命令

  1. 用Debug的R命令查看、改变CPU寄存器的内容
  2. 用Debug的D命令查看内存中的内容
  3. 用Debug的E命令改写内存中的内容
  4. 用Debug的U命令将内存中的机器指令翻译成汇编指令
  5. 用Debug的T命令执行一条机器指令
  6. 用Debug的A命令以汇编指令的格式在内存中写入一条机器指令
